[0:00 — HOOK]

Every day, millions of elderly people miss medications, forget to log their vitals,
and struggle to reach their caregivers when they need help most.
This agent was built to fix that — a personal AI concierge that manages medications,
tracks wellness, and coordinates care automatically.

[0:20 — WHAT IT IS]

The Elderly Care Assistant is a conversational AI agent built on Google's Agent
Development Kit — ADK 2.0.
It belongs to the Healthcare & Wellness track.
It combines multiple specialized sub-agents, a security layer, a live SQLite database
via MCP, and a human-approval gate — all working together in one unified workflow.

[1:00 — ARCHITECTURE]

(show assets/workflow_diagram.png)

Let me walk you through the flow, node by node.

Every request starts at the START node and immediately passes through the
security_checkpoint — this is the gatekeeper.
It scrubs personally identifiable information like phone numbers, SSNs, and emails.
It detects prompt injection attempts.
It blocks requests that try to override medical protocols.

If the input is flagged as unsafe, it routes to format_security_response — and the
conversation ends safely with an advisory message.

If the input passes — it moves to classify_request.
classify_request reads the intent: is this about medications, wellness, or caregiver
updates?

For medication and wellness queries, the request goes to the orchestrator.
The orchestrator delegates to two specialist sub-agents:
  - medication_advisor — retrieves medication schedules and checks drug interactions
    using the MCP SQLite tools.
  - wellness_monitor — logs vitals like blood pressure, glucose, and mood, and
    retrieves historical summaries.

For caregiver notifications, the request goes to caregiver_approval.
(pause)
This is where the human-in-the-loop happens.
The workflow pauses and asks the patient: "Do you confirm sending this update to your
caregiver?"
Only after a clear YES does it proceed to build_caregiver_message and
caregiver_updater.

All paths converge at final_output, which formats and returns the response to the user.

[3:00 — SECURITY & MCP]

The security_checkpoint uses regex filters to redact PII in real time.
It also maintains an audit trail of every blocked request.

The MCP Server exposes five SQLite tools:
  - log_health_metric — logs vitals with threshold alerts
  - get_medication_schedule — retrieves daily dose plans
  - add_medication_reminder — registers new medications with interaction checks
  - get_wellness_summary — queries health trends over time
  - check_drug_interaction — flags dangerous medication pairings

All data stays local. No PII leaves the device.
